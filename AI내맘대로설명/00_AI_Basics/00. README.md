# AI Basics

# AI를 처음 이해하기 위한 흐름 정리

요즘은 정말 어디를 가도 AI 이야기가 나온다.

- ChatGPT
    
- Claude
    
- Gemini
    
- RAG
    
- AI Agent
    
- 벡터 검색
    
- MCP
    

그런데 막상 이런 단어들을 들으면 생각보다 헷갈리는 경우가 많다.

특히:

```text
왜 갑자기 벡터가 나오지?
왜 검색 시스템이 필요하지?
왜 AI를 통제해야 하지?
```

같은 궁금증이 자연스럽게 생긴다.

이 문서는 그런 흐름들을:

```text
처음부터 순서대로
```

이해하기 위해 작성되었다.

---

# 이 문서의 목표

이 문서는 단순히:

```text
기술 용어 설명
```

만 하는 문서가 아니다.

오히려:

```text
왜 이런 기술이 등장했는가?
```

를 이해하는 데 더 집중한다.

예를 들어:

- 왜 AI는 검색 시스템이 필요할까?
    
- 왜 벡터 검색이 등장했을까?
    
- 왜 RAG가 필요할까?
    
- 왜 AI를 통제하려고 할까?
    

같은 흐름이다.

즉:

```text
문제
↓
한계
↓
그래서 등장한 기술
```

순서로 이해하는 것을 목표로 한다.

# 핵심 흐름 한 번에 보기

현재 많은 AI 시스템은 대략 다음 흐름으로 발전하고 있다.

```text
AI
↓
LLM
↓
LLM의 한계
↓
검색 시스템 필요
↓
Vector
↓
RAG
↓
AI Agent
↓
Guardrail
↓
실제 AI 운영 시스템
```

즉 이제 AI는 단순 챗봇을 넘어서:

```text
검색 시스템 + 업무 자동화 + 운영 플랫폼
```

방향으로 발전하고 있다.

# 마무리

AI는 생각보다:

```text
하나의 기술
```

이라기보다,

```text
여러 기술과 시스템이 연결된 구조
```

에 가깝다.

이 문서가:

```text
각 기술이 왜 등장했고
어떻게 연결되는가
```

를 이해하는 데 도움이 되었으면 좋겠다!! 짠!
