# 02. How LLM Works

# LLM은 왜 사람처럼 말할 수 있을까?

이전 글에서 설명했듯이 현재 AI는:

```text
패턴 기반 확률 예측 시스템
```

에 가깝다.

그런데 여기서 자연스럽게 이런 궁금증이 생긴다.

```text
그럼 왜 사람처럼 대화가 가능하지?
```

이번에는 그 이유를 아주 쉽게 설명해보려고 한다.

---

# LLM이란?

요즘 ChatGPT 같은 AI를 보통:

```text
LLM (Large Language Model)
```

이라고 부른다.

뜻을 풀어보면:

- Large = 엄청 크고
    
- Language = 언어를 다루고
    
- Model = 학습된 AI 모델
    

즉:

```text
언어를 학습한 거대한 AI
```

정도로 이해하면 된다.

대표적으로:

- GPT
    
- Claude
    
- Gemini
    

같은 AI들이 여기에 해당한다.

---

# LLM은 무엇을 학습할까?

LLM은 인터넷에 있는 엄청난 데이터를 학습한다.

예를 들어:

- 책
    
- 뉴스
    
- 블로그
    
- 코드
    
- 논문
    
- 대화 데이터
    

같은 것들이다.

이 데이터를 계속 읽으면서:

```text
어떤 단어 다음에
어떤 단어가 자주 나오는가
```

를 학습한다.

---

# 예시로 이해해보기

예를 들어:

```text
오늘 점심은 햄버거를 먹고
```

라는 문장이 있다면 사람은 자연스럽게:

- 싶다
    
- 왔다
    
- 배불렀다
    

같은 말을 예상할 수 있다.

AI도 비슷하다.

다만 AI는 사람처럼 생각한다기보다:

```text
과거 데이터에서 가장 가능성이 높은 다음 단어
```

를 계산하는 방식에 가깝다.

---

# 그런데 왜 이렇게 자연스러울까?

핵심은 학습량이다.

사람이 평생 읽는 글보다 훨씬 많은 데이터를 AI는 학습한다.

그래서:

```text
사람들이 보통 어떻게 말하는가
```

를 매우 많이 알고 있다.

예를 들어:

- 사과는 과일이다
    
- 고양이는 동물이다
    
- 비가 오면 우산을 쓴다
    

같은 패턴을 엄청나게 많이 본다.

그래서 답변이 자연스럽게 보이는 것이다.

---

# AI는 문장을 조금씩 생성한다

많은 사람들이 AI가 답변 전체를 한 번에 만든다고 생각한다.

하지만 실제로는:

```text
한 단어씩
조금씩 생성
```

하는 방식에 가깝다.

예를 들어:

```text
안녕하세요
오늘은
AI에 대해
설명해보겠습니다
```

같이 순서대로 만들어간다.

즉:

```text
현재까지 만들어진 문장
```

을 보고 다음 단어를 계속 예측하는 구조이다.

---

# Token이라는 개념

여기서 중요한 개념이 하나 나온다.

바로:

```text
Token
```

이다.

AI는 문장을 그대로 이해하기보다:

```text
잘게 나눈 조각
```

처럼 처리한다.

예를 들어:

```text
안녕하세요 저는 개발자입니다
```

라는 문장은 내부적으로 여러 개의 Token으로 나뉘어 처리된다.

AI 비용 계산에서도 이 Token이 많이 사용된다.

보통:

- 입력 Token
    
- 출력 Token
    

양으로 비용을 계산한다.

---

# Context Window는 무엇인가?

AI는 대화를 무한히 기억하지 못한다.

한 번에 볼 수 있는 양에 제한이 있다.

이 범위를 보통:

```text
Context Window
```

라고 부른다.

쉽게 말하면:

```text
AI의 현재 작업 공간 크기
```

같은 느낌이다.

---

# 그래서 오래 대화하면 문제가 생기기도 한다

예를 들어:

- 너무 긴 문서
    
- 너무 긴 대화
    
- 너무 많은 자료
    

를 넣으면 AI가:

- 앞 내용을 잊거나
    
- 중요한 정보를 놓치거나
    
- 이상한 답을 하는 경우
    

가 생길 수 있다.

그래서 실제 AI 시스템에서는:

```text
필요한 정보만 골라서 넣는 작업
```

이 굉장히 중요하다.

---

# 그런데 AI는 최신 정보를 항상 아는 것은 아니다

많은 사람들이 오해하는 부분 중 하나다.

LLM은 보통:

```text
특정 시점까지 학습
```

된 데이터 기반으로 동작한다.

즉 학습 이후 새로 생긴 정보는 모를 수 있다.

예를 들어:

- 오늘 뉴스
    
- 방금 올라온 문서
    
- 회사 내부 자료
    

같은 것은 기본적으로 모르는 경우가 많다.

---

# 그래서 다음 기술들이 등장한다

여기서부터:

- 검색 시스템
    
- 벡터 검색
    
- RAG
    

같은 기술들이 등장한다.

왜냐하면:

```text
AI 혼자서는
최신 정보와 회사 데이터를 잘 모른다
```

는 문제가 있기 때문이다.

즉 이후 기술들은 대부분:

```text
LLM의 한계를 보완하기 위해
```

등장한 구조라고 이해하면 된다.

---

# 정리

현재의 LLM은:

```text
엄청난 양의 언어 데이터를 학습하고
다음 단어를 예측하며
자연스럽게 문장을 생성하는 시스템
```

이라고 볼 수 있다.

그리고 여기서 다음 단계로:

```text
왜 AI는 검색 시스템이 필요할까?
```

라는 주제로 이어지게 된다.
