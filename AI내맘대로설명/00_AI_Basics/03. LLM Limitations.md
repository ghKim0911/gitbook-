# 03. LLM Limitations

# LLM은 왜 한계가 있을까?

이전 글에서 설명했듯이 LLM은:

```text
엄청난 데이터를 학습해서
다음 단어를 예측하는 시스템
```

에 가깝다.

그런데 실제로 AI를 사용해보면 가끔 이상한 모습을 볼 수 있다.

예를 들어:

- 틀린 말을 자신 있게 하거나
    
- 존재하지 않는 정보를 만들어내거나
    
- 아까 했던 말을 잊어버리거나
    
- 회사 문서를 모른다고 하거나
    
- 이상한 방향으로 대화를 끌고 가는 경우
    

가 있다.

이번에는 왜 이런 문제가 발생하는지 알아보려고 한다.

---

# AI는 사실을 "검증"하는 시스템이 아니다

많은 사람들이:

```text
AI = 모든 것을 알고 있는 존재
```

처럼 생각한다.

하지만 실제 AI는:

```text
그럴듯한 문장을 생성
```

하는 시스템에 더 가깝다.

즉:

```text
이 말이 진짜인가?
```

를 검증한다기보다,

```text
이 문장이 자연스러운가?
```

를 기준으로 답변하는 경우가 많다.

---

# 그래서 환각(Hallucination)이 발생한다

예를 들어 AI가:

- 존재하지 않는 논문
    
- 존재하지 않는 URL
    
- 존재하지 않는 회사 정책
    

을 그럴듯하게 말하는 경우가 있다.

이 현상을 보통:

```text
Hallucination (환각)
```

이라고 부른다.

이건 AI가 거짓말을 한다기보다:

```text
확률적으로 자연스러운 문장
```

을 생성하려다 발생하는 문제에 가깝다.

---

# AI는 생각보다 기억력이 약하다

많은 사람들이 AI가 모든 대화를 계속 기억한다고 생각한다.

하지만 실제로는:

```text
한 번에 볼 수 있는 양
```

에 제한이 있다.

이 제한을 보통:

```text
Context Window
```

라고 부른다.

쉽게 말하면:

```text
AI의 현재 작업 공간 크기
```

같은 개념이다.

---

# 그래서 긴 대화에서 문제가 생긴다

예를 들어:

- 너무 긴 회의록
    
- PDF 수백 페이지
    
- 아주 긴 채팅 기록
    

같은 것을 한 번에 넣으면 AI가:

- 앞 내용을 잊거나
    
- 중요 정보를 놓치거나
    
- 서로 다른 내용을 섞어버리는 경우
    

가 생길 수 있다.

---

# AI는 최신 정보를 항상 알지 못한다

이 부분도 매우 중요하다.

LLM은 보통:

```text
특정 시점까지의 데이터
```

를 학습한다.

즉:

- 오늘 뉴스
    
- 방금 수정된 문서
    
- 회사 내부 자료
    
- 최신 정책
    

같은 것은 기본적으로 모르는 경우가 많다.

그래서 AI에게:

```text
우리 회사 휴가 정책 알려줘
```

라고 물어보면,

실제 회사 문서를 읽은 것이 아니라:

```text
일반적인 회사 정책 패턴
```

으로 답변할 가능성이 있다.

---

# AI는 방향을 잘못 잡기도 한다

이것도 실제로 자주 발생한다.

예를 들어:

```text
코끼리를 찾아줘
```

라고 요청했다고 해보자.

그런데 조건이 애매하면 AI는:

- 실제 동물 코끼리
    
- 코끼리 장난감
    
- 코끼리 캐릭터
    
- 코끼리 관련 뉴스
    

같은 것들을 모두 관련 있다고 판단할 수 있다.

즉:

```text
조건이 모호하면
AI도 모호하게 행동
```

하게 된다.

---

# AI는 반복 행동도 할 수 있다

AI는 생각보다:

```text
같은 행동 반복
```

을 자주 한다.

예를 들어:

- 같은 검색 반복
    
- 같은 사이트 반복 접근
    
- 비슷한 질문 반복
    
- 의미 없는 루프
    

같은 일이 발생할 수 있다.

특히 AI Agent처럼:

```text
AI가 직접 행동하는 구조
```

에서는 이런 문제가 더 커진다.

---

# 그래서 AI를 그대로 믿으면 위험할 수 있다

실제 서비스에서는 다음 문제가 발생할 수 있다.

- 비용 폭주
    
- 잘못된 정보 제공
    
- 무한 루프
    
- 관련 없는 자료 검색
    
- 위험한 행동 수행
    

그래서 실제 AI 시스템은 단순히:

```text
LLM 하나만 사용하는 구조
```

로 잘 만들지 않는다.

---

# 그래서 여러 보완 기술이 등장한다

이 문제들을 해결하기 위해:

- 검색 시스템
    
- 벡터 검색
    
- RAG
    
- Rule
    
- Guardrail
    
- Approval
    
- Monitoring
    

같은 구조들이 등장한다.

예를 들어:

```text
AI가 회사 문서를 모르네?
```

↓

```text
그러면 검색해서 같이 넣어주자
```

↓

```text
그게 RAG
```

같은 흐름이다.

---

# 중요한 점

현재 AI는 매우 강력한 도구이지만:

```text
완벽하게 스스로 판단하는 존재
```

는 아니다.

그래서 실제로는 대부분:

```text
AI + 검색 + 규칙 + 사람 검토
```

구조를 함께 사용한다.

---

# 정리

LLM의 대표적인 한계는:

- 사실 검증이 약함
    
- 환각 발생 가능
    
- 기억 제한 존재
    
- 최신 정보 부족
    
- 방향 이탈 가능
    
- 반복 행동 가능
    

같은 것들이다.

그리고 이후 기술들은 대부분:

```text
이 한계를 보완하기 위해
```

등장한 구조라고 이해하면 된다.

다음 글에서는:

```text
왜 AI에는 검색 시스템이 필요한가?
```

를 알아보려고 한다.
