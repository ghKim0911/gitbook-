# 05. Vector Concept

# 벡터(Vector)란 무엇인가?

이전 글에서는:

```text
왜 AI에 검색 시스템이 필요한가?
```

를 알아봤다.

그런데 여기서 또 새로운 문제가 생긴다.

예를 들어 사람이:

```text
휴가 정책 알려줘
```

라고 질문했다고 해보자.

그런데 문서에는:

```text
연차 규정
```

이라고 적혀 있을 수도 있다.

사람은:

- 휴가
    
- 연차
    
- 쉬는 날
    

같은 말들이 비슷한 의미라는 것을 자연스럽게 이해한다.

그런데 컴퓨터는 이런 의미를 원래 잘 이해하지 못한다.

그래서 등장한 개념이:

```text
벡터(Vector)
```

이다.

---

# 컴퓨터는 숫자를 좋아한다

컴퓨터는 기본적으로:

```text
숫자 계산
```

을 잘한다.

반대로:

```text
문장의 의미
```

같은 것은 그대로 이해하기 어렵다.

그래서 AI 시스템은 문장을:

```text
숫자 묶음
```

으로 변환해서 사용하기 시작했다.

---

# 문장을 숫자로 바꾼다

예를 들어:

```text
고양이
```

라는 단어를 숫자로 바꾸면:

```text
[0.12, -0.82, 0.55, ...]
```

같은 형태가 된다.

그리고:

```text
강아지
```

도 다른 숫자 형태로 변환된다.

```text
[0.10, -0.79, 0.52, ...]
```

이 숫자 묶음을 보통:

```text
벡터(Vector)
```

라고 부른다.

---

# 중요한 건 숫자 자체가 아니다

사실 중요한 건:

```text
숫자 모양
```

그 자체가 아니다.

진짜 중요한 것은:

```text
비슷한 의미끼리
가까운 위치에 배치
```

된다는 점이다.

예를 들어:

- 고양이
    
- 강아지
    
- 햄스터
    

같은 단어들은 서로 비슷한 동물이라 위치도 가까워진다.

반대로:

- 데이터베이스
    
- 자동차
    
- 회계
    

같은 전혀 다른 의미는 멀리 떨어진다.

---

# 쉽게 말하면 "의미 좌표" 같은 개념이다

벡터를 아주 쉽게 표현하면:

```text
문장의 의미를 좌표처럼 저장
```

하는 느낌에 가깝다.

즉 AI는:

```text
의미가 비슷한가?
```

를 숫자 거리 계산으로 판단한다.

---

# 그래서 의미 기반 검색이 가능해진다

예를 들어 사용자가:

```text
쉬는 날 규정 알려줘
```

라고 질문했다고 해보자.

문서에는:

```text
연차 정책
```

이라고 적혀 있어도,

AI는 벡터를 이용해서:

```text
둘이 비슷한 의미
```

라고 판단할 수 있다.

즉:

```text
단어 자체
```

보다:

```text
의미 자체
```

를 비교하는 검색이 가능해진 것이다.

---

# 기존 검색과의 차이

기존 검색은 보통:

```text
키워드 기반 검색
```

이었다.

즉:

```text
문서 안에 정확히 같은 단어가 있는가?
```

를 기준으로 찾았다.

하지만 벡터 검색은:

```text
의미가 비슷한가?
```

를 기준으로 찾는다.

그래서:

- 표현 방식이 달라도
    
- 단어가 조금 달라도
    
- 문장이 달라도
    

비슷한 내용을 찾을 수 있다.

---

# 임베딩(Embedding)이라는 개념

여기서 자주 등장하는 용어가 하나 더 있다.

바로:

```text
Embedding
```

이다.

Embedding은 쉽게 말하면:

```text
문장을 벡터로 변환하는 과정
```

이다.

즉:

```text
문장
↓
숫자 벡터 변환
```

과정을 의미한다.

보통:

- 벡터 = 결과물
    
- 임베딩 = 변환 과정
    

처럼 이해하면 쉽다.

---

# 벡터는 어디에 저장할까?

이렇게 만들어진 벡터는 보통:

```text
벡터 DB(Vector Database)
```

에 저장한다.

예를 들어:

- pgvector
    
- Pinecone
    
- Milvus
    
- Weaviate
    

같은 시스템들이 사용된다.

이 DB들은:

```text
비슷한 벡터를 빠르게 찾는 기능
```

에 특화되어 있다.

---

# 현재 AI 검색의 핵심 기술 중 하나

현재 AI 시스템에서는 벡터 검색이 매우 중요하다.

왜냐하면:

```text
사람의 의미 기반 검색 방식
```

을 컴퓨터가 흉내내기 시작했기 때문이다.

즉 단순 키워드 검색보다:

- 더 자연스럽고
    
- 더 유연하고
    
- 더 관련성 높은 검색
    

이 가능해졌다.

---

# 정리

벡터(Vector)는 쉽게 말하면:

```text
문장의 의미를 숫자 좌표처럼 표현한 것
```

이다.

그리고 이를 통해 AI는:

```text
단어 자체가 아니라
의미 자체를 비교
```

할 수 있게 되었다.

그리고 이 벡터 검색과 LLM을 결합한 구조가 바로:

```text
RAG
```

이다.

다음 글에서는:

```text
RAG는 어떻게 동작하는가?
```

를 알아보려고 한다.
